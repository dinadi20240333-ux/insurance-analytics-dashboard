

# Insurance Website User Behaviour Analytics Dashboard

## Project Overview

This project develops an interactive **Streamlit dashboard** to analyse insurance website user behaviour and marketing channel performance.

The dashboard provides insights into:
- User engagement
- Marketing channel effectiveness
- Revenue performance
- Insurance quote generation
- Policy purchase behaviour
- Customer conversion patterns

The purpose of this dashboard is to help an insurance company understand how users from different marketing channels interact with their website and identify opportunities to improve business performance.

---

## Project Objectives

The main objectives of this project are:

- Analyse website users based on marketing channels and device categories.
- Identify channels generating the highest website traffic.
- Evaluate revenue contribution from different marketing sources.
- Analyse the relationship between insurance quotes and policy purchases.
- Provide an interactive dashboard for exploring customer behaviour.

---

## Dashboard Features

### Key Performance Indicators (KPIs)

The dashboard displays:

- Total Website Users
- Total Revenue
- Total Insurance Quotes
- Total Policies Purchased
- Conversion Rate


### Interactive Filters

Users can filter dashboard results by:

- Marketing Channel
- Device Category


### Visualisations

The dashboard includes:

Users by Marketing Channel  
Revenue by Marketing Channel  
Device Category Distribution  
User Engagement vs Policy Purchases  
Insurance Conversion Funnel

## Installation and Setup

### 1. Clone the repository

```
git clone https://github.com/yourusername/insurance-analytics-dashboard.git
```

### 2. Install required libraries

```
pip install -r requirements.txt
```

### 3. Run the Streamlit application

```
streamlit run first_app.py
```
